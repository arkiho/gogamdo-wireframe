## 접근 1: Blueprint Wireframe (건축 도면 스타일)

**Design Movement:** 건축 블루프린트(청사진) 미학

**Core Principles:**
1. 건축 도면의 격자와 치수선을 모티프로 한 와이어프레임
2. 청색 계열의 모노크롬 팔레트로 블루프린트 감성 극대화
3. 각 요소에 어노테이션(주석)을 달아 기획 의도를 명확히 전달

**Color Philosophy:** 짙은 네이비 배경에 밝은 시안/화이트 라인으로 건축 도면의 정밀함과 전문성을 표현. 인테리어 회사의 정체성과 자연스럽게 연결.

**Layout Paradigm:** 격자 기반의 정밀한 레이아웃. 각 섹션이 도면의 한 구역처럼 구분되며, 치수선과 주석이 함께 표시.

**Signature Elements:** 
1. 점선 격자 배경과 치수선 표기
2. 각 UI 요소 옆에 붙는 어노테이션 박스 (기능 설명)

**Interaction Philosophy:** 섹션 클릭 시 해당 페이지의 상세 와이어프레임으로 전환. 어노테이션에 마우스 오버 시 상세 설명 표시.

**Animation:** 페이지 전환 시 도면이 펼쳐지듯 슬라이드. 요소가 연필로 그려지듯 순차적으로 나타남.

**Typography System:** Noto Sans KR (본문) + JetBrains Mono (어노테이션/기술 텍스트)

## 접근 2: Sketch Pad Wireframe (스케치 패드 스타일)

**Design Movement:** 핸드 드로잉/스케치 미학

**Core Principles:**
1. 손으로 그린 듯한 스케치 느낌의 와이어프레임
2. 연필/마커 텍스처로 초기 기획 단계의 생동감 표현
3. 포스트잇, 화살표 등 실제 기획 회의실의 요소 활용

**Color Philosophy:** 크림색 종이 배경에 연필 회색 라인. 핵심 CTA나 강조 요소만 빨간/파란 마커 색상으로 하이라이트.

**Layout Paradigm:** 자유로운 스케치북 레이아웃. 약간의 기울어짐과 불규칙성으로 핸드메이드 느낌 연출. 포스트잇 형태의 메모가 곳곳에 배치.

**Signature Elements:**
1. 손그림 스타일의 UI 요소 (둥글둥글한 박스, 물결 라인)
2. 노란 포스트잇 메모로 기능 설명

**Interaction Philosophy:** 포스트잇 클릭 시 상세 설명 팝업. 페이지 간 이동은 스케치북 넘기듯 전환.

**Animation:** 요소가 연필로 그려지는 드로잉 애니메이션. 포스트잇이 붙여지는 효과.

**Typography System:** Nanum Pen Script (제목/메모) + Noto Sans KR (본문 설명)

## 접근 3: Technical Wireframe (테크니컬 다이어그램 스타일)

**Design Movement:** 시스템 다이어그램 / UI 프로토타이핑 미학

**Core Principles:**
1. 깔끔한 회색 톤의 전문적인 와이어프레임
2. 실제 UI 프로토타이핑 도구(Figma/Balsamiq)와 유사한 표현
3. 각 요소의 크기, 간격, 기능이 명확히 표기된 실용적 문서

**Color Philosophy:** 밝은 회색(#F5F5F5) 배경에 다크 그레이 라인. 인터랙티브 요소는 블루 계열로 하이라이트. 어노테이션은 오렌지/그린으로 구분.

**Layout Paradigm:** 정확한 12컬럼 그리드 시스템 기반. 각 섹션에 명확한 경계선과 라벨. 사이드바에 페이지 네비게이션 트리 표시.

**Signature Elements:**
1. 크로스해치(X) 패턴의 이미지 플레이스홀더
2. 컬러 코딩된 어노테이션 시스템 (파랑=링크, 주황=기능, 초록=콘텐츠)

**Interaction Philosophy:** 좌측 사이드바에서 페이지를 선택하면 우측에 해당 와이어프레임이 표시. 각 요소에 호버 시 상세 스펙 표시.

**Animation:** 최소한의 부드러운 전환. 요소 호버 시 미세한 확대와 어노테이션 팝업.

**Typography System:** Pretendard (본문) + IBM Plex Mono (어노테이션/스펙)

</idea>
